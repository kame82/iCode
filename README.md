# README

## サービス概要

- Web 制作における HTML、CSS、JavaScript などのコードの保管管理が出来るサービス。
- これまで実装したアイテム（ボタンやハンバーガーメニューなど）のコードを保存しておき、必要時に呼び起こして閲覧することで、実装時の時短が可能となる。
- 記載したコードはリアルタイム編集とプレビューのリアルタイム同期により、即時確認可能 →code pen のような使い方が可能。

## このサービスへの思い・作りたい理由

- Web 制作におけるコード管理の課題を解決し、開発者にとってより便利で使いやすいサービスを作りたいという思いから開発しました。Notion は汎用性の高いメモアプリですが、以下の課題があります。

  - コードのリアルタイムプレビューができない
  - コード共有機能が限定的
  - 初心者にとって操作が難しい

- そこで、以下の点を差別化ポイントとして、Notion よりも優位なサービスを目指します。
  - コード編集とプレビューのリアルタイム同期
  - 公開・非公開設定による柔軟なコード共有
  - 初心者でも使いやすいシンプルな操作性

## ユーザー層について

- このサービスは、以下のユーザー層を想定しています。

  - プログラミング初学者: 独学でプログラミングを学習している人や、初めて Web 制作に挑戦する人

    - コードの書き方を学びながら、実際に使えるコードをストックしたい
    - 分からない部分をすぐに確認できる機能が欲しい

  - Web コーダー: 効率的にコード開発を進めたい人や、チームで開発している人
    - 繰り返し使用するコードを簡単に共有・管理したい
    - コードレビュー機能で、より良いコードを書きたい

## サービスの利用イメージ

- このサービスは、以下の様な利用シーンを想定しています。
  - コーディング中に、過去のコードをすぐに参照したい
  - デザインカンプから HTML、CSS、JavaScript コードを効率的に生成したい
  - チームメンバーとコードを共有し、レビューや修正を行いたい
  - 公開可能なコードを他の開発者と共有し、情報交換や学びの場としたい
  - デザインのアイデアをメモ代わりに記録しておき、後からコード化したい

## ユーザーの獲得について

- 以下の施策を実施することで、ユーザーを獲得していきます。

  - SNS での情報発信: ターゲット層が利用している SNS を活用し、サービスの機能や魅力を積極的に発信します。
  - Web 制作スクール生への紹介: Web 制作スクールに通う学生や卒業生向けに、サービスを紹介する機会を設けます。
  - オープン β 版の提供: ユーザーに実際にサービスを体験してもらい、フィードバックをもとに改善を行います。

## サービスの差別化ポイント・推しポイント

- コード編集とプレビューのリアルタイム同期
- 公開・非公開設定による柔軟なコード共有
- 初心者でも使いやすいシンプルな操作性
- コードレビュー機能による、より良いコードの開発

- 競合調査

  - notion

    - **Pros**: 汎用性の高いメモアプリ。サービスとしてもしっかりしており、普段使用する分には困らない
    - **Cons**: 自己完結 ⇒ 自分のコードしか見れないため共有性に欠ける。notion 初心者にとっては操作が難しい。
    - **差別化ポイント**:Notion では基本的に他人のコードストックを閲覧出来ないため、他人のコードが見れる点を差別化ポイントとできる。また、Notion でコード管理する際はどうしてもコードのみの管理となってしまいがちである。画像などによる実装イメージがわかる点も差別化出来ると思う。

  - codestock: 個人開発アプリ

    - **Pros**: プライベート機能、いいね機能、お気に入り機能は使い勝手が良かった。検索上位に出るため SEO 対策もしっかりしている。
    - **Cons**: デザインが全体的に「青 × 黒」で暗い。写真やイメージが無いためコードを使用した時のイメージが分かりづらい。最近更新されていない。閲覧出来るのはコードのみで、サイト上からデザインの確認は出来ない。
    - **差別化ポイント**: 実装イメージがつくように画像添付やコードイメージのリアルタイム反映。AI によるリファクタリング機能を実装することで差別化可能。

  - codepen

    - **Pros**: シェアの高いコードアプリ。ユーザー数は多い。コード一覧のデザインが見やすい。
    - **Cons**: 日本語対応していないため、初学者の使用障壁が高い。無料プランの場合、書いたコードがすべて公開されてしまう。
    - **差別化ポイント**: 無料でコードの非公開機能が使用可能。日本語対応。

  - plunker

    - **Pros**: コード編集画面の使いやすさが最も高く、codepen よりも見やすい。
    - **Cons**: コード一覧が見にくく、検索性や操作性が悪い。日本語対応していないため、初学者の使用障壁が高い。
    - **差別化ポイント**: 無料でコードの非公開機能が使用可能。日本語対応。

  - Runstant: 個人開発アプリ

    - **Pros**: サーバー処理が極限まで抑えられており、保存・実行が高速化されている。Matter.js や Three.js などの実行も可能
    - **Cons**: コード一覧のデザインに難があり操作感が悪い。
    - **差別化ポイント**: codepen のような見やすいデザインで実装。

  - v0.dev: AI がテキストからデザインを考えてコーディングしてくれるサービス

    - **Pros**: コーディングの負担が大幅に低減する。
    - **Cons**: デザインが AI 頼りになってしまう。
    - **差別化ポイント**: 現状の Web 制作は、デザイナーが考えたデザインをコーダーがコーディングして形にしている。vo.dev のような AI による Web 制作が主流となれば、コーダーのしごとは無くなり、当然このサービスは必要なくなってしまう。現時点では AI による web 制作はまだ課題を抱えているため、数年はコーダーの仕事が無くなることはないと思う。差別化ポイントとしては、レシピアプリのような使用感を実現することで差別化出来ると考えており、デザイナーはデザイン案の参考に、コーダーはコーディング時の参考に。といった使い方が出来る。これを実現するには、情報の蓄積と引き出しやすさがポイントになると考えている。

## 機能の実装方針予定

- コードプレビュー:

  - iframe を用いたリアルタイムプレビュー
  - コード実行結果プレビュー
  - レスポンシブデザインプレビュー

- セキュリティ:

  - Sanitize gem を用いた悪意のある script タグの削除
  - SQL インジェクション対策
  - クロスサイトスクリプティング対策
